Optical Proximity Correction (OPC)

Description

Lithography is the process of printing tiny circuit patterns onto silicon wafers. Light travels through a mask at different angles, controlled by the illumination source, onto a light-sensitive surface that records the pattern. The problem is that at the scales modern chips operate at, light bends and interferes with itself, causing patterns to blur, round off and distort by the time they reach the wafer (see this video). What you design is never quite what gets printed. Take a simple example, using a desired pattern directly as the mask: Using Fourier optics, we can describe this behaviour precisely. Given an illumination source and mask, the model can predict the resulting intensity pattern and final resist profile on the wafer. The challenge is going the other way: given a desired resist profile, find the illumination and mask that produce it. For a model this complex, that inversion is far from straightforward, and this project tackles it using a surrogate UNet FiLM neural network and gradient-based optimisation.

Simulating

Using the Fourier optics model, we simulate the full imaging process step by step. The mask is transformed into frequency space, where each frequency represents a spatial detail of the pattern. The illumination source controls which angles of light hit the mask, and for each of those angles a pupil filter removes the finest details that the physical lens doesn't have the resolution to reproduce, similar to how a camera lens with a low aperture produces a blurry image. All these contributions are summed and transformed back to give the intensity pattern that lands on the wafer. Finally, the photoresist, which is the light-sensitive material on the wafer, responds to this intensity with a threshold effect: regions that receive enough light get exposed and regions that don't stay unexposed, giving the final binary result.

The simulator is accurate but slow, and more importantly not differentiable, meaning we can't use it directly to optimise the mask and illumination through gradient descent. That's where the surrogate neural network comes in.

Surrogate Neural Network

We can train a neural network to approximate the simulator by generating mask and illumination pairs and using the Fourier optics model to compute the ground truth wafer intensity and resist profile for each one. Convolutional layers are particularly well suited for this task because the mask is inherently spatial, and the behaviour at each pixel depends strongly on its local neighbourhood.

The illumination is handled differently. Unlike the mask, the illumination source is a global property of the optical system, affecting how every part of the mask gets resolved on the wafer equally. Feeding it in as just another input channel and processing it with convolutions would not reflect this. Instead, we use Feature-wise Linear Modulation (FiLM): the illumination is encoded into a compact vector, which is then used to dynamically scale and shift the feature maps at each level of the network. This is similar to a standard layer in a neural network, but rather than having fixed learned parameters, the scale and shift values are determined dynamically from the illumination input. This allows the network to adapt its feature extraction to different illumination conditions without changing the spatial structure of the mask features.

The architecture used is a UNet, as shown below, a natural choice for image-to-image prediction tasks. The network consists of two main parts: an encoder and a decoder. The encoder processes the input mask through a sequence of convolutional layers and downsampling operations. At each stage it extracts increasingly abstract spatial features while reducing the resolution of the representation, allowing the network to capture larger spatial context and global structure in the mask pattern. At the centre of the network is a bottleneck representation that contains a compact summary of the mask features. From this compressed representation, the decoder gradually reconstructs a high-resolution output by applying upsampling layers followed by convolutional blocks. At each stage the spatial resolution increases while the representation becomes more detailed again.

Skip connections link each encoder layer to the corresponding decoder layer. These connections pass high-resolution feature maps directly to later stages of the network, providing a shortcut for fine spatial detail that would otherwise be lost during downsampling. This helps the model preserve edge information and local structure in the predicted wafer intensity and resist profile.

Using this UNet-FiLM, the network predicts wafer intensity and resist profiles that preserve mask details while adapting to illumination. An example output, which compares it to the ground truth simulation, is shown below. Once trained, gradients can flow through the surrogate neural network, meaning we can use it to work backwards from a desired resist profile to find the mask and illumination that produce it.

Data Generation

To train the surrogate neural network, we need a large and diverse dataset of mask, illumination and simulated output pairs. We start with a set of 4000 base circuit patterns, but that alone wouldn't give the network enough variety to generalise well. During optimisation, the network will encounter mask and illumination combinations it has never seen before. If those combinations fall outside the training distribution, the network's predictions become unreliable, and the optimiser ends up exploiting these blind spots rather than finding physically meaningful solutions. To fix this, we apply a range of augmentations to generate thousands of variations including dilation, erosion, boundary noise, corner rounding and random feature additions, producing masks that could resemble OPC-corrected patterns. For the illumination, we generate random source patterns by placing Gaussian modes at different positions in frequency space, and blending them with random weights. Each mask and illumination pair is passed through the simulator to generate ground truth intensity and resist profiles, giving us a dataset of around 50k training samples to start with.

Gradient-Based Optimisation

With a trained surrogate neural network, we can now optimise the mask and illumination jointly. Because the network is differentiable, we can compute gradients of the resist prediction with respect to both inputs, and use those gradients to find adjustments that bring the predicted output closer to the desired pattern. The optimisation starts from the target pattern itself as the initial mask, with a randomly sampled illumination. To avoid getting stuck in a local minima, we apply a Gaussian blur to the mask that gradually decreases over iterations, starting coarse and becoming sharper. This lets the optimiser find the right global structure before committing to fine details. A binary phase at the end pushes mask values toward 0 or 1, since the masks are binary. The results of an example run are shown below: the target pattern, a baseline simulation using the target directly as a mask, the optimised mask and illumination, and finally the network prediction alongside the ground truth simulator output. While the surrogate neural network (7th img) predicts a good result, the ground truth simulator (8th img) tells a different story. The optimiser found a solution that looks convincing to the network, but sits outside the region it was trained on, where its predictions can no longer be trusted. This is where active learning comes in.

Active Learning

While the optimiser might return a solution with a low loss according to the surrogate, this does not guarantee the result is physically accurate. When simulated with the ground truth simulator, the printed result can look quite different, as the optimiser has pushed the inputs into regions where the surrogate's predictions become unreliable.

To tackle this, active learning is used. Instead of treating the training dataset as fixed, we use the optimisation process itself to generate extra training data. Snapshots of the mask taken throughout each optimisation run are collected and simulated with the ground truth simulator to get accurate labels. These new samples are added to the training set and training continues, gradually improving the surrogate in exactly the regions the optimiser cares about. Each round of optimisation generates better training data for the next. Over 20 rounds, adding around 2500 new samples each time, the surrogate becomes more reliable in the regions that matter for optimisation.
